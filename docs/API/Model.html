ALLOW_INCOMPLETE

This symbol can be used to create models with missing "required" fields. Note that this value is inherited, and so can be referenced from your own model's name rather than using Model.ALLOW_INCOMPLETE. E.g. 

  class Something extends Model {
    // ...
  }

  Something.create(undefined, Something.ALLOW_INCOMPLETE);

(async) save()

Save this model to file, either by explicitly specifying a filename, or by relying on the Model's __meta.filename attribute, which must indicate which key path to use as "primary key" equivalent. Note that the filename is just the name, without any directory or extension information. The directory will be automatically determined based on the model name, and all models are saved as json, using the .json extension.
